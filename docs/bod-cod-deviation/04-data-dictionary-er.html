Data Dictionary และ ER Diagram

ฐานข้อมูลสำหรับรายงานค่าความคลาดเคลื่อน BOD/COD Online และแบบแจ้งผลการตรวจสอบ ส่วนกลาง/ส่วนภูมิภาค

DB และ Workflow แบบรายงานผล แบบแจ้งผลส่วนกลาง แบบแจ้งผลส่วนภูมิภาค ER Diagram 6 แบบ
ขอบเขตการออกแบบ

รายงาน 1 ฉบับเลือกพารามิเตอร์ได้เพียง 1 รายการ คือ BOD หรือ COD เท่านั้น ข้อมูลโรงงานถูก snapshot ลงรายงานตอนส่ง เพื่อให้การออกเอกสารย้อนหลังไม่เปลี่ยนตาม master data ที่อาจแก้ภายหลัง

1. ER Diagram

PK = Primary Key FK = Foreign Key 1:N = รายงานหนึ่งฉบับมีได้หลายรายการ 1:1 = รายงานหนึ่งฉบับมีผลตรวจแจ้งผลหนึ่งชุด

2. Relationship Summary

Parent Child Cardinality คำอธิบาย
bod_cod_deviation_reports.id bod_cod_deviation_measurements.report_id 1:N หนึ่งรายงานมีแถวผลตรวจได้หลายแถว แต่ทุกแถวต้องเป็น parameter เดียวกับที่เลือกในรายงาน
bod_cod_deviation_reports.id bod_cod_deviation_reviews.report_id 1:1 หนึ่งรายงานมีผลตรวจเพื่อออกแบบแจ้งผลหนึ่งชุด
bod_cod_deviation_reports.id bod_cod_deviation_attachments.report_id 1:N หนึ่งรายงานต้องมีไฟล์แนบตามประเภทที่กำหนด
bod_cod_deviation_reports.id bod_cod_approval_steps.report_id 1:N สร้าง step ตามพื้นที่: CENTRAL = 3 step, REGIONAL = 2 step
bod_cod_deviation_reports.id bod_cod_approval_events.report_id 1:N เก็บประวัติ action ทุกครั้งของรายงาน
bod_cod_approval_steps.id bod_cod_approval_events.step_id 1:N event บางรายการผูกกับ step เฉพาะ เช่น approve หรือ edit_after_approval
