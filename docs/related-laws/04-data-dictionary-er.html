8. Business Rules

  • ประชาชนและผู้ประกอบการเห็นเฉพาะรายการที่ laws.status = 'published', laws.is_public = 1 และ laws.deleted_at IS NULL
  • ปุ่มดาวน์โหลดต้องเลือกไฟล์จาก law_files.is_current = 1 ของกฎหมายนั้น
  • เมื่อเจ้าหน้าที่แก้ไขไฟล์ ให้เพิ่มแถวใหม่ใน law_files และเพิ่ม version_no ห้าม overwrite ไฟล์เดิม
  • กฎหมายหนึ่งรายการควรมีไฟล์ current ได้ไม่เกิน 1 ไฟล์
  • อนุญาตชนิดไฟล์ตาม policy เช่น pdf, doc, docx และควรจำกัดขนาดไฟล์สูงสุด
  • ทุกการสร้าง แก้ไข เปลี่ยนไฟล์ เผยแพร่ ยกเลิกเผยแพร่ และลบ ต้องบันทึกใน law_audit_logs
  • ทุกการดาวน์โหลดควรบันทึกใน law_download_logs เพื่อดูสถิติและตรวจสอบย้อนหลัง
  • การลบรายการกฎหมายควรใช้ soft delete ผ่าน deleted_at เพื่อรักษาประวัติไฟล์และ audit

10. API ที่ควรมี

MethodEndpointใช้โดยคำอธิบาย
GET/api/v1/lawspublicรายการกฎหมายที่เผยแพร่แล้ว
GET/api/v1/laws/:idpublicรายละเอียดกฎหมาย

POST/api/v1/admin/lawsofficerเพิ่มข้อมูลกฎหมาย
PATCH/api/v1/admin/laws/:idofficerแก้ไข metadata เช่น ชื่อ เลขเอกสาร วันที่ประกาศ หมวดหมู่
