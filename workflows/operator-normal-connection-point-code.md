# Operator Normal Connection Point Code Workflow

Status: Ready for implementation

## Trigger

คำขอเชื่อมต่อใหม่แบบปกติของผู้ประกอบการผ่านการยืนยันการเชื่อมต่อ และ backend ต้องออกรหัสให้จุดตรวจวัดที่ยังไม่มี `pointCode`.

## Goal

ให้ backend ออกรหัสจุดตรวจวัดของคำขอปกติแบบเรียงลำดับ แยกลำดับตามระบบ โดยเริ่ม CEMS ที่ `S2001` และ WPMS ที่ `W2001`.

## Scope

- ใช้เฉพาะคำขอปกติของผู้ประกอบการ (`submissionSource` ไม่ใช่ `OFFICER_DIRECT_API`).
- CEMS ใช้ `S` ตามด้วยเลขอย่างน้อย 4 หลัก: `S2001`, `S2002`, ...
- WPMS ใช้ `W` ตามด้วยเลขอย่างน้อย 4 หลัก: `W2001`, `W2002`, ...
- CEMS และ WPMS มีลำดับแยกจากกัน.
- หากคำขอมีหลายจุด ระบบจองเลขติดกันตามลำดับจุดในคำขอ.
- การจองเลขต้องปลอดภัยเมื่อมีหลายคำขอยืนยันพร้อมกัน.
- คำขอ `ADD_PARAMETER` ไม่ออกรหัสใหม่.
- การเชื่อมต่อโดยเจ้าหน้าที่โดยตรงยังใช้ `measurementPoints[].pointCode` ที่เจ้าหน้าที่กรอกเอง และไม่เข้าสู่ลำดับนี้.
- ไม่แก้ไขหรือแปลงรหัสเดิมที่มีอยู่ก่อน deploy รวมถึงรหัส WPMS รูปแบบ `Pxxxx`.
- ลำดับใหม่เริ่มขั้นต่ำที่ `2001`; หากมีรหัส prefix เดียวกันซึ่งมีเลขสูงกว่าอยู่แล้ว ให้ต่อจากเลขสูงสุดเพื่อหลีกเลี่ยงรหัสซ้ำ.
- รหัส `Pxxxx` เดิมไม่ถูกนำมาคำนวณลำดับ `Wxxxx` ใหม่.

## Public Interface

ไม่มี endpoint ใหม่และ request payload ไม่เปลี่ยน พฤติกรรมที่สังเกตได้คือ เมื่อ status action ของคำขอปกติทำให้ backend ออกรหัสจุดตรวจวัด จุด CEMS แรกของลำดับใหม่ได้ `S2001` และจุด WPMS แรกได้ `W2001`; API รายละเอียดคำขอและ API จุดตรวจวัดที่เชื่อมต่อแล้วต้องคืนรหัสเดียวกัน.

## Failure And Consistency Rules

- การออกรหัสและการเปลี่ยนสถานะอยู่ใน transaction เดียวกัน.
- ห้ามออกรหัสซ้ำแม้เกิด concurrent requests.
- ถ้าการบันทึกสถานะหรือจุดตรวจวัดล้มเหลว ต้อง rollback การจองเลขตาม transaction behavior เดิม.

## TDD Behaviors

1. การยืนยันคำขอ CEMS ปกติครั้งแรกในลำดับใหม่ออก `S2001` และครั้งถัดไปออก `S2002`.
2. การยืนยันคำขอ WPMS ปกติครั้งแรกในลำดับใหม่ออก `W2001` และครั้งถัดไปออก `W2002`.
3. ลำดับ CEMS และ WPMS ไม่กระทบกัน.
4. Direct Connect ไม่ถูกสร้างรหัสอัตโนมัติและเก็บรหัสที่เจ้าหน้าที่ส่งมาเหมือนเดิม.

## Resolved Decisions

- คงรหัสเดิมทุกค่าไว้โดยไม่ migrate; การเปลี่ยนแปลงมีผลเฉพาะรหัสที่ระบบออกใหม่หลัง deploy.
- ตัวนับเริ่มขั้นต่ำที่ `2000` เพื่อให้รหัสแรกเป็น `S2001` หรือ `W2001`.
- หากข้อมูลเดิมมี `Sxxxx` หรือ `Wxxxx` ที่สูงกว่า ระบบต้องออกเลขถัดจากค่าสูงสุด.
- การล้างข้อมูลก่อนขึ้นระบบจริงเป็นงาน deployment แยก ไม่ใช่หน้าที่ของ migration นี้.
