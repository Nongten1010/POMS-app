# Operator Normal Connection Point Code Workflow

Status: Implemented

## Trigger

คำขอเชื่อมต่อใหม่แบบปกติของผู้ประกอบการผ่านการอนุมัติแบบ และ backend ต้องออกรหัสให้จุดตรวจวัดที่ยังไม่มี `pointCode`.

## Goal

ให้ backend ออกรหัสจุดตรวจวัดของคำขอปกติแบบเรียงลำดับ แยกลำดับตามระบบ โดย CEMS ใช้ `SNNNN` และ WPMS ใช้ `WNNNN`.

## Scope

- ใช้เฉพาะคำขอปกติของผู้ประกอบการ (`submissionSource` ไม่ใช่ `OFFICER_DIRECT_API`).
- CEMS ใช้ `S` ตามด้วยลำดับอย่างน้อย 4 หลัก เริ่มขั้นต่ำที่ `S2001`.
- WPMS ใช้ `W` ตามด้วยลำดับอย่างน้อย 4 หลัก เริ่มขั้นต่ำที่ `W2001`.
- CEMS และ WPMS มีลำดับแยกจากกันและไม่เริ่มใหม่เมื่อเปลี่ยนปี.
- หากคำขอมีหลายจุด ระบบจองเลขติดกันตามลำดับจุดในคำขอ.
- การจองเลขต้องปลอดภัยเมื่อมีหลายคำขอยืนยันพร้อมกัน.
- คำขอ `ADD_PARAMETER` ไม่ออกรหัสใหม่.
- การเชื่อมต่อโดยเจ้าหน้าที่โดยตรงยังใช้ `measurementPoints[].pointCode` ที่เจ้าหน้าที่กรอกเอง และไม่เข้าสู่ลำดับนี้.
- ไม่แก้ไขหรือแปลงรหัสเดิมที่มีอยู่ก่อน deploy.
- หากมีรหัส `S...` หรือ `W...` ที่สูงกว่า sequence ที่บันทึกไว้ ให้ต่อจากเลขสูงสุดเพื่อหลีกเลี่ยงรหัสซ้ำ.
- รหัสรูปแบบอื่น เช่น `Pxxxx`, `CEMS-NNNN/YYYY` และ `WEMS-NNNN/YYYY` ไม่ถูกนำมาคำนวณลำดับใหม่.

## Public Interface

ไม่มี endpoint ใหม่และ request payload ไม่เปลี่ยน พฤติกรรมที่สังเกตได้คือ เมื่อ status action ของคำขอปกติทำให้ backend ออกรหัสจุดตรวจวัด จุด CEMS แรกได้ขั้นต่ำ `S2001` และจุด WPMS แรกได้ขั้นต่ำ `W2001`; API รายละเอียดคำขอและ API จุดตรวจวัดที่เชื่อมต่อแล้วต้องคืนรหัสเดียวกัน.

## Failure And Consistency Rules

- การออกรหัสและการเปลี่ยนสถานะอยู่ใน transaction เดียวกัน.
- ห้ามออกรหัสซ้ำแม้เกิด concurrent requests.
- ถ้าการบันทึกสถานะหรือจุดตรวจวัดล้มเหลว ต้อง rollback การจองเลขตาม transaction behavior เดิม.

## TDD Behaviors

1. การอนุมัติคำขอ CEMS ปกติครั้งแรกออก `S2001` และจุดถัดไปออก `S2002`.
2. การอนุมัติคำขอ WPMS ที่มี sequence ล่าสุด 2002 ออก `W2003`.
3. ลำดับ CEMS และ WPMS ไม่กระทบกัน.
4. เมื่อเปลี่ยนปี ลำดับจุดเดิมยังเดินต่อและไม่เริ่มใหม่.
5. Direct Connect ไม่ถูกสร้างรหัสอัตโนมัติและเก็บรหัสที่เจ้าหน้าที่ส่งมาเหมือนเดิม.
6. Route ยังอ่าน annual point-code เดิมได้เพื่อ backward compatibility แต่ generator ใหม่ไม่ออก annual point-code.

## Resolved Decisions

- คงรหัสเดิมทุกค่าไว้โดยไม่ migrate; การเปลี่ยนแปลงมีผลเฉพาะรหัสที่ระบบออกใหม่หลัง deploy.
- ตัวนับเก็บแยกตาม `systemType` และมีค่าขั้นต่ำ 2000 เพื่อให้รหัสแรกเป็น `S2001` หรือ `W2001`.
- หากข้อมูลเดิมมีรหัส `S...`/`W...` ที่สูงกว่า sequence ระบบต้องออกเลขถัดจากค่าสูงสุด.
- การล้างข้อมูลก่อนขึ้นระบบจริงเป็นงาน deployment แยก ไม่ใช่หน้าที่ของ migration นี้.
